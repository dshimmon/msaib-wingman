Mission 001 — Foundation
Objective

Establish the foundation for MSAIB Wingman by creating a professional development environment and initializing the project's Git repository.

Deliverables
Installed and configured VS Code
Connected GitHub
Installed Git, Python, Node.js, and npm
Created the Wingman project structure
Initialized the Git repository
Created the initial README
Published the repository to GitHub
Engineering Concepts
Finder vs. VS Code vs. Terminal
Projects are folders
Git repositories
GitHub
Commits
Local vs. remote repositories
Key Lessons
Git tracks the history of a project, not just files.
GitHub is an online copy of the repository, not Git itself.
Every meaningful milestone deserves its own commit.
Interview Takeaway

Be able to explain the difference between Git and GitHub and describe the standard Git workflow:
Write → Add → Commit → Push.

Mission 002 — First Heartbeat
Objective

Transform MSAIB Wingman into a runnable Python application using a professional project structure.

Deliverables
Created a virtual environment
Added a .gitignore file
Created the src directory
Created main.py
Executed the first Python program
Published the second commit to GitHub
Engineering Concepts
Virtual environments
Source code organization
Hidden project folders
.gitignore
Running Python from the terminal
Key Lessons
Every project should have its own virtual environment.
Generated files should not be committed to Git.
The editor is where software is written.
The terminal is where software is executed.
Interview Takeaway

Explain why professional Python projects use virtual environments and .gitignore, and why isolating dependencies improves reliability.
