# Mission 003 — First Contact

**Mission Call Sign:** First Contact

**Status:** ✅ Complete

---

## Objective

Transform MSAIB Wingman from a static application into an interactive application capable of communicating with its pilot.

---

## Deliverables

- Added user input using `input()`
- Introduced variables to store user responses
- Personalized Wingman's responses using f-strings
- Improved the application's console interface
- Successfully executed Wingman's first interactive conversation

---

## Engineering Concepts

- User input
- Variables
- Strings
- f-strings
- Program flow
- Refactoring

---

## Key Lessons

- Interactive software begins by receiving input from the user.
- Variables allow programs to remember information during execution.
- F-strings provide a clean way to combine text with variable values.
- Refactoring improves code readability without changing functionality.
- Software becomes significantly more useful once it can react to user input.

---

## Interview Takeaway

Describe how information flows through an application:

1. The application prompts the user.
2. The user's response is stored in a variable.
3. The application uses that variable to generate a personalized response.

Also explain why f-strings are preferred for readable, maintainable output.

Mission 003 marked the moment Wingman became interactive.
