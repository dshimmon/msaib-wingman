SUPPORTED RETRIEVAL PATHS
=========================

1. DOCUMENT-TEXT RETRIEVAL

Example:
Orientation

Process:
Bare-topic guardrail
→ text_search_terms
→ knowledge.py
→ matching text evidence


2. STRUCTURED-RECORD RETRIEVAL

Example:
What are the Fall Module A courses and times?

Process:
OpenAI interpretation
→ record type: course_schedule
→ exact filter: module = Mod A
→ deterministic record matching


3. CONCEPT-MEMORY RETRIEVAL

Example:
Where has MSAIB Curriculum appeared?

Process:
OpenAI interpretation
→ memory_search_terms
→ concept_retrieval.py
→ persistent occurrence memory


RESPONSIBILITY BOUNDARY
=======================

OPENAI

- Interprets natural-language requests
- Produces structured retrieval instructions
- Reasons over organized context
- Communicates the final answer


WINGMAN

- Creates structured knowledge
- Preserves source metadata
- Stores concepts and records
- Maintains persistent concept memory
- Executes retrieval instructions deterministically
- Builds organized context
- Preserves supporting sources


MISSION 016 ARCHITECTURAL OUTCOME
=================================

Before Mission 016, Wingman created enriched knowledge
but primarily searched raw text.

After Mission 016, Wingman can use three distinct knowledge
representations:

- Document evidence
- Structured records
- Persistent concept memory

The representation selected depends on a structured plan
produced by OpenAI and executed deterministically by Wingman.
